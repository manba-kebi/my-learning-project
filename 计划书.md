## 八、项目方案：最适合你的 C++/AI 项目

项目名称建议：

`LiteLLM-Serve`

中文描述：

基于 C++17 的本地大模型推理服务

### 项目目标

做一个能在本地加载模型、对外提供接口、支持基础并发处理、能做日志和压测的 C++ 推理服务。

### 为什么这个项目适合你

- 它以 C++ 为主，不需要你先转 Python 后端。
- 它和 AI 热点有关，但不要求你训练模型。
- 它能同时证明你具备 C++、Linux、并发、工程化能力。
- 它写进简历后，比普通课程项目更像真实工作。

### 推荐技术栈

- 语言：C++17
- 构建：CMake
- 推理后端：llama.cpp
- 网络接口：cpp-httplib
- JSON：nlohmann/json
- 日志：spdlog
- 配置：yaml-cpp
- 测试：GoogleTest
- 部署：Docker
- 调试与性能：GDB、perf

### 推荐目录结构

- `include/`
- `src/`
- `tests/`
- `configs/`
- `tools/`
- `scripts/`
- `README.md`
- `CMakeLists.txt`
- `Dockerfile`

### 核心模块

- `engine`：封装模型加载和推理接口
- `server`：提供 HTTP API
- `scheduler`：处理请求队列、线程池、并发控制
- `session`：管理多轮会话上下文
- `common`：日志、配置、指标、工具函数
- `tools/bench`：压测和性能统计

### 功能拆分

#### V1 必做

- 能加载本地模型
- 提供单轮问答接口
- 有配置文件
- 有日志
- 有 `/health` 健康检查
- README 可以指导别人跑起来

#### V2 加分

- 支持流式输出
- 支持多轮上下文
- 支持 `temperature`、`top_p`、`max_tokens`
- 有异常处理
- 有基础测试
- 有 Docker 部署

#### V3 冲刺

- 有请求队列
- 有线程池
- 有最大并发控制
- 有超时处理
- 有性能指标接口
- 有 benchmark 和性能报告

### 这个项目最重要的不是功能数量，而是这四件事

- 结构是否清晰
- 是否有工程化意识
- 是否有并发和调试痕迹
- 是否有性能数据

## 九、简历怎么写这个项目

### C++ 开发岗写法

项目名称：

`LiteLLM-Serve | 基于 C++17 的本地大模型推理服务`

可写成下面这种表述：

- 使用 C++17 设计并实现本地大模型推理服务，完成推理引擎封装、HTTP 接口、配置管理与日志模块开发。
- 封装模型加载和推理调用流程，支持基础会话管理与参数控制，降低上层接口与底层推理库耦合。
- 基于线程池和请求队列实现并发控制、超时处理和拒绝策略，提升服务在高并发场景下的稳定性。
- 编写 benchmark 工具，对 QPS、平均延迟、P95 延迟、tokens/s 等指标进行统计与分析。
- 使用 CMake、GoogleTest、Docker 完成项目构建、测试与部署，提升工程可维护性。

### 测试开发岗写法

项目名称：

`LiteLLM-Serve | AI 推理服务测试与稳定性工程实践`

可写成下面这种表述：

- 基于 C++17 搭建本地 AI 推理服务，围绕接口正确性、并发稳定性与性能表现设计测试场景。
- 编写自动化脚本和压测工具，覆盖参数校验、异常输入、会话管理、并发请求与超时处理等场景。
- 对服务的 QPS、延迟、资源占用等指标进行监控与分析，定位并复现稳定性问题。
- 结合配置文件和 Docker 实现可复现运行环境，提升测试验证效率。

### 简历写法的硬规则

- 不要写“熟练掌握”。
- 不要写空话，如“提高了代码质量”但没有解释怎么做。
- 要写你做了什么、解决了什么、测了什么。
- 有数据就写数据，没有数据就先测再写。

## 十、你提到的这些知识点，都是 C++ 的吗

不是全部都属于“C++ 语法”，但它们都属于 C++ 岗非常高频的核心能力。

可以分成 4 类。

### 第一类：C++ 语言核心能力

- RAII
- 智能指针
- move 语义

这三类是非常典型的现代 C++ 内容，属于你必须补上的东西。

### 第二类：C++ 并发与标准库能力

- 线程与锁

这部分既属于 C++，也属于并发编程基础。常见内容包括：

- `std::thread`
- `std::mutex`
- `std::lock_guard`
- `std::unique_lock`
- 条件变量
- 死锁与锁粒度

### 第三类：操作系统与网络基础

- 进程线程模型
- TCP/IP
- GDB
- 性能分析基础

这些不是“C++ 语法”，但几乎所有像样的 C++ 岗位都会问到。因为 C++ 常常被用于接近系统层、接近性能瓶颈的位置，所以企业会默认你需要懂这些。

### 第四类：工程工具能力

- CMake
- Git

这两项也不是 C++ 语法，但它们是做 C++ 项目离不开的工具能力。

### 最重要的判断

你可以这样理解：

RAII、智能指针、move 语义是“语言能力”。

线程与锁、进程线程模型、TCP/IP、GDB、性能分析是“系统与调试能力”。

CMake、Git 是“工程化能力”。

三者合起来，才是招聘方眼里的“可用的 C++ 候选人”。

## 十一、这些知识点应该怎么学

你的学习方式不能再是“看一堆视频，然后感觉自己会了”。

更有效的方法是四步法。

### 四步学习法

1. 先理解概念，不求一次学很深，但必须知道它解决什么问题。
2. 写最小 demo，代码量控制在 30 到 100 行。
3. 主动制造 bug，再用调试工具定位。
4. 把它放进你的项目里，形成真实使用场景。

只有做到第 4 步，这个知识点才算真的进入你的能力里。

### 你的学习比例建议

- 30% 时间看资料和理解概念
- 70% 时间写 demo、调 bug、改项目

如果你花很多时间“看懂了”，但没有写过最小 demo，也没有在项目中用过，这种学习很容易在面试时崩掉。

### 你的输出标准

每学一个知识点，至少留下下面三样东西中的两样：

- 最小 demo
- 笔记或 bug 复盘
- 项目中的实际应用

## 十二、每个知识点的具体学习方法

### 1. RAII

你要学什么：

- 为什么 C++ 强调对象生命周期
- 为什么资源要绑定到对象作用域
- 为什么 RAII 能减少内存泄漏和异常路径泄漏

你要写什么：

- 文件句柄封装类
- 简单互斥锁守卫类
- 一个对比 demo：手动释放资源 vs RAII 自动释放

你要达到什么水平：

能清楚说出：

“RAII 的核心不是智能指针，而是把资源的获取和释放绑定到对象生命周期上，构造时拿到资源，析构时自动释放资源。”

### 2. 智能指针

你要学什么：

- `unique_ptr` 的独占所有权
- `shared_ptr` 的共享所有权
- `weak_ptr` 解决循环引用
- 为什么优先使用 `make_unique` 和 `make_shared`

你要写什么：

- 把你以前一个使用 `new/delete` 的小程序改成智能指针版本
- 做一个 `shared_ptr` 循环引用和 `weak_ptr` 解决问题的 demo

你要避免什么：

- 不要把智能指针当作“更高级的裸指针”
- 不要到处滥用 `shared_ptr`

### 3. move 语义

你要学什么：

- 左值、右值、右值引用
- 为什么要有 move
- move 和 copy 的区别
- 移动后对象处于什么状态

你要写什么：

- 自己实现一个简单的动态数组或字符串类
- 分别实现拷贝构造和移动构造
- 打印日志观察复制和移动发生的时机

你要达到什么水平：

能说出：

“move 的目的不是语法炫技，而是在资源可转移时减少不必要拷贝。”

### 4. 线程与锁

你要学什么：

- 线程创建与回收
- 临界区
- 互斥锁
- 死锁
- 条件变量
- 生产者消费者模型

你要写什么：

- 多线程打印 demo
- 线程安全计数器
- 生产者消费者队列
- 简单线程池

你要达到什么水平：

至少能自己实现一个基础线程池，并能解释锁的作用、死锁的成因、如何缩小锁粒度。

### 5. 进程线程模型

你要学什么：

- 进程和线程的区别
- 用户态和内核态的基本概念
- 上下文切换的大致成本
- 多进程与多线程的适用场景

你要做什么：

- 在 Linux 环境下观察进程和线程
- 看懂 `ps`、`top`、`htop`、`pstree` 的基本输出

### 6. TCP/IP

你要学什么：

- TCP 和 UDP 的区别
- 三次握手、四次挥手
- socket 编程基本流程
- 阻塞与非阻塞
- 为什么服务端常见 `epoll`

你要写什么：

- TCP echo server/client
- 最小 HTTP 服务

你要达到什么水平：

能画出一个最基础的网络服务调用链：

`socket -> bind -> listen -> accept -> recv/send -> close`

### 7. GDB

你要学什么：

- 断点
- 单步执行
- 查看栈帧
- 查看变量
- 回溯调用链

你要做什么：

- 故意写一个段错误程序
- 用 GDB 定位问题
- 故意写一个多线程问题，观察调试过程

你要达到什么水平：

遇到崩溃，不是第一反应“重写一遍”，而是先定位。

### 8. CMake

你要学什么：

- `add_library`
- `target_include_directories`
- `target_link_libraries`
- Debug 和 Release 的概念

你要做什么：

- 让项目支持独立构建和测试

你要达到什么水平：

能自己从零写一个中小型项目的 `CMakeLists.txt`，而不是只会复制模板。

### 9. Git

你要学什么：

- `commit`
- `log`
- `branch`

- `push`

你要做什么：

你要达到什么水平：

### 10. 性能分析基础

你要学什么：

- 什么是 CPU 时间、系统时间、墙钟时间
- 如何做最基础的性能测量
- 为什么需要 benchmark

你要做什么：

- 用 `time` 测程序耗时
- 用 `perf stat` 看基础性能统计
- 用 `perf record` 和 `perf report` 看热点
- 记录优化前后数据

你要达到什么水平：

能说出：

“我不是主观觉得它快了，而是我测过了。”
