## 2026年5月6日

- 今天跟着博主学习，主要学习到了就是怎么将对一个套接字socket的相关操作进行封装成类的成员函数，为了分开客户端与服务端的操作，还通过封装好的 Socket 基类，派生出 Server_socket 和 Client_socket 子类。
- 还学习到了在TCP中IO阻塞与非阻塞的区别。**IO阻塞就是如果没得到回应，进程会阻塞并让出 CPU，进程进入睡眠状态，直到数据就绪才被唤醒。而非阻塞IO就是问一下，如果没得到回应就返回会返回错误（通常是 `EWOULDBLOCK` 或 `EAGAIN`），不会让进程等待，程序可以稍后重试或处理其他任务。**
- 还学习了一些相关操作，比如`set_reuseaddr`、`set_linger`、`set_keepalive`、设置发送缓冲区（SO_SNDBUF）的大小、设置接收缓冲区（SO_RCVBUF）的大小
  - `set_reuseaddr` (SO_REUSEADDR)：允许服务器在重启时立即绑定到之前关闭的端口，即使该端口还处于 `TIME_WAIT` 状态（通常 2MSL 等待）。如果不设置，重启服务器可能会因为端口被占用而绑定失败。
  - `set_linger` 控制 `closesocket`（或 `close`）时的行为：
    - 如果 `l_onoff = 0`（默认），`closesocket` 立即返回，系统会尝试优雅关闭（**发送剩余数据，但不会等待确认**）。
    - 如果 `l_onoff = 1` 且 `l_linger` 为 0，则**直接发送 RST 并丢弃发送缓冲区中的数据**（强制关闭）。
    - 如果 `l_onoff = 1` 且 `l_linger > 0`，**则 `closesocket` 会最多阻塞 `l_linger` 秒，尽力发送完缓冲区数据并正常四次挥手。**
  - `set_keepalive` 启用 TCP 的保活机制：当连接空闲一段时间后，系统会自动向对端发送探测包。如果对端无响应（多次重试后），连接会被关闭，适用于检测连接是否意外中断（如断电、网络故障）。

## 2026年5月18日

- 今天大部分时间都是在上课，整软件工程的作业。
- 所以今天稳扎稳打，有复习了一遍RAII、智能指针、move语义、thread
  - RAII
    - RAII是资源管理的重要思想，将资源生命周期绑定到对象生命周期上
    - RAII在构造函数中获取资源，在析构函数中释放资源，这样在对象退出作用域时一定会回收资源
    - RAII不只是管理内存，还可以管理文件、线程、socket、mutex、数据库链接等系统资源
    - 如果一个类独占资源，那么应该禁止拷贝，避免两个对象重复释放同一个资源
    - 在mini_kv_server中，Socket与SocketRuntime中都是RAII思想的体现
  - 智能指针
    - 智能指针是用RAII管理动态对象生命周期的标准库工具
    - unique_ptr 独占所有权，不能拷贝，只能被移动
    - shared_ptr 共享所有权，通过控制块维护引用计数信息，引用计数为0时释放对象
    - weak_ptr 不拥有对象，不增加引用计数，通常用于观察对象或者打破循环引用，
    - 在 mini_kv_server 中，客户端 socket 通过 shared_ptr 捕获进线程池任务，是为了让连接对象在线程处理期间保持存活
  - move语义
    - 移动语义用于将一个对象的资源所有权转移到另一个对象，避免昂贵拷贝
    - std::move本身不移动资源，只是将表达式转换为可以被移动的右值表达式
    - 真正发生资源转移的是移动构造函数或移动赋值函数
    - 拥有独占资源的类通常要禁用拷贝，并实现移动构造和移动赋值
    - 在 mini_kv_server 中的socket 就是move-only类型的，移动时转移socket handle,并把源对象置为无效
  - thread
    - std::thread 表示一个正在运行或者曾经运行过的线程对象
    - 线程启动后，如果对象析构时仍然joinable()，那么程序会调用 std::terminate()
    - 所以每个线程最终必须join()或者detach()，工程中更常用join()
    - std::thread 参数默认会拷贝，如果需要传引用需要std::ref()
    - mini_kv_server 的线程池内部保存多个std::thread，析构时统一join()
